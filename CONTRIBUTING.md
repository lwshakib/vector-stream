# Contributing to Vector Stream

First off, thank you for considering contributing to Vector Stream! It's people like you that make Vector Stream such a great tool.

## How Can I Contribute?

### Reporting Bugs

- **Check if the bug has already been reported** by searching on GitHub under [Issues](https://github.com/lwshakib/vector-stream/issues).
- If you can't find an open issue addressing the problem, [open a new one](https://github.com/lwshakib/vector-stream/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample** or an **executable test case** demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements

- **Check if the enhancement has already been suggested.**
- [Open a new issue](https://github.com/lwshakib/vector-stream/issues/new) and describe the enhancement you'd like to see.

### Pull Requests

1. **Fork the repo** and create your branch from `main`.
2. **Install dependencies** using `bun install`.
3. **Make your changes**. If you've added code that should be tested, add tests.
4. **Ensure the test suite passes**.
5. **Format your code** with the project's standards.
6. **Submit a pull request**!

## Style Guide

- We use **TypeScript** for both frontend and backend.
- Follow the existing code style and naming conventions.
- Use meaningful commit messages.

## Development Setup

See the [README.md](README.md) for instructions on setting up the project locally.

## Community

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).
